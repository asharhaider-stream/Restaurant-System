# Restaurant Management System

A modern **Flutter-based restaurant application** designed with **scalability and modular architecture** in mind.  
The system currently focuses on the **Customer Mobile App** with full UI flows, structured for future **backend integration, AI automation, and multi-device support**.

## Features
- Customer-facing **menu browsing and order placement**  
- Clean and **scalable MVVM architecture** for each feature (auth, restaurant browsing, explore, profile)  
- Reusable **widgets, services, and Riverpod state management**  
- Fully **mocked data-driven UI** for development and learning  
- Designed for **future AI chatbot integration, voice assistant, and automation**  

## Tech Stack
- **Flutter (Dart)** for mobile development  
- **MVVM Architecture** with features structured as models, view, view_model, and repository  
- **Riverpod** for state management  
- **API-ready design** for future backend integration  
- **SRC folder** for shared resources: assets, routing, services, theme, utils, widgets
